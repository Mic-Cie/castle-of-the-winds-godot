Please create a Window-mode game in Godot.
The game should allow a character to move around a map of tiles.
Create a controllable player object, but use design patterns that would eventually make it possible to play in a multiplayer mode.

Draw a map from tiles given in tiles.png (move the file to some assets folder).
Displayed game area should consist of 32x32 tiles. The game view should not be scalable - the only thing that changes in the game view with resolution or window size should be how much of the game area is drawn.
Notes about tiles.png:
tiles.png consists of 7x10 (column x row) tile.
Tile 1-1 is a wall.
Tile 1-3 is a floor

For the hero character use first tile from monsters.png (move the file to some assets folder)
Notes about monsters.png:
14x5 tiles
Tile 1-1 is the hero

Maps should be created by calling a level generator. The only implemented level generator for now is "fixed" - it creates a predefined map.
For now create a simple map filled with wall tiles and some interconnected rooms.

The map should be large enough to showcase scrolling.
Map can be scrolled in two ways:
- by following the hero, however - it should not scroll with every hero's movement. There should be some tolerance of distance of the player from the border of currently draw area, before the game recenters on the player. THe game should check this with every player's movement - it should not be done in real time.
- by widgets on game area border, like Windows' scrolling widgets, or Godot's scrolling widgets, if exist.

There should be no move animations. THe characters should be simply redrawn whenever they move to another tile.
