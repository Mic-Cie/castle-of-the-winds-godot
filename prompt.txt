Please add time.
Game should start at zero time.
Every step of a character has a time cost (base cost: 5 seconds). Search also has a time cost (Base cost: 10 seconds).
These activities are two different kinds:
Step happens immediately - character cannot do anything else for the equivalent of time cost.
Search happens after the equivalent of time cost. 

Time should be able to flow in two ways, depending on game mode.
In single player mode, the game waits for the player's input. After player does some action, all other actions that happen are calculated (since there are no characters for now, there will be nothing to do). Render the game after the action has been completed or interrupted.

In multi player mode, the game ticks every second and renders at every tick. If a character does an immediate action - they cannot do anything else until enough game ticks happen. If a character does an action like Search, that happens AFTER some time, the character can interrupt this action anytime with another action.

Time should be associated with the game, not a character. Show time in the bottom_panel.
